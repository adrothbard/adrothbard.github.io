PWAs

The function of a PWA is to provide the user with an UI/UX interaction that feels like a traditional app, but instead uses a web browser to deploy. This app can be installed on the users desktop via a browser, and although this page is a simple device to change the background image, there are much more robust app programming options available using this method. In order to accomplish this taks we need to add a web app manifest file and a service worker.


JSON: The Critical Element

For this page, a JSON manifest file has been created for a basic PWA. The items in this JSON file include the app name, description, and multiple icons of various sizes to be associated with the app to name a few. The PWA absolutely relies on the use of the JSON for the app to work, and without it, the simple PWA would break. The user will be able to see the option to install the application when using the menu of the browser - see the image shown here when a web application with PWA features is presented in the Chrome browser, shows and instal option in the menu of the browser. The user will also see the icon to download the application in the navigation bar of the browser. This page actually employs two different JavaScript files in addition to the JSON file - one of which calls the service worker script into action, while simultaneously calling the function of changing the background images. The second piece of JavaScript used for this page is the service worker script itself, which performs the multiple functions described on the above page.


Service Workers and a Simple Test

We can use a JavaScript service worker file to help build our PWA, but also service workers can also be employed for background tasks, such as providing continuity for the user if an Internet connection is cut off. The images and text of the page are preserved while any connectivity issues can be resolved.

Button Test

Above are two buttons that allow the user to change the background image of this page. What the user probably doesn't know is that the developer has also installed a JavaScript Service Worker that opens the caches object with the name that was given in the variable CACHE_VERSION. Viewing the Cache Storage in Chrome Developer tools indicates all of the elements available to the user even when browsing offline, including the background images and basic html of this page.
